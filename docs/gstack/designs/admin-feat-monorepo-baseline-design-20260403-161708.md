## Problem Statement

perfect-panel 的 API client 生成依赖外部仓库 ppanel-docs 的 Swagger JSON，且使用维护活跃度下降的 @umijs/openapi 生成器。每次 API 变更需三步手动操作（改 .api → 更新 ppanel-docs → 前端 openapi 生成），monorepo 内前后端代码已在同一仓库却仍需外部中转。现有生成的 client 需要手动包装 useQuery/queryKey，47 个组件文件中存在大量重复模式。

## What Makes This Cool

改一个 `.api` 文件，跑一个命令：Go handler + OpenAPI spec + TypeScript SDK + TanStack Query hooks 全部自动更新。去掉 axios 依赖，拥抱原生 fetch。AI 助手可以直接从本地 spec 理解所有接口。

## Terminology

- **gopure** — 仓库内的自定义 API 定义处理二进制（`server/generate/gopure-*`），等价于 goctl 但为本项目定制
- **goctl-openapi** — 独立 Go 插件（`github.com/jayvynl/goctl-openapi`），被 gopure 以 `-plugin` 参数调用，负责从 `.api` 文件生成 OpenAPI 3.0.3 JSON spec

## Constraints

- Server 的 `.api` 文件和 gopure 代码生成流程不变
- @hey-api/openapi-ts v0.94+ 替换 @umijs/openapi
- client-fetch（非 client-axios），去掉 axios 依赖
- OpenAPI spec 格式为 JSON，提交到 git
- Redocly lint 校验，需禁用 no-empty-servers 等规则（goctl-openapi 不生成 servers 字段）
- swagger_node.api 不纳入（节点通信接口，前端不调用）
- 本方案不支持 Windows 开发环境（gopure 二进制和 shell 脚本仅覆盖 macOS/Linux）

## Premises

1. goctl-openapi 从 .api 文件生成 OpenAPI 3.0.3 JSON spec，取代 ppanel-docs 外部仓库
2. @hey-api/openapi-ts + client-fetch 替换 @umijs/openapi，去掉 axios 依赖
3. TanStack Query 插件自动生成 queryOptions/mutationOptions
4. 一步到位在同一 PR，PR 内按功能分 commit
5. API.* 全局命名空间通过 re-export 兼容层桥接
6. 拦截器从 axios 迁移到 fetch interceptor，skipErrorHandler 用 silentClient 替代

### 1.3 request.ts（现有拦截器）

```typescript
// apps/admin/utils/request.ts — 变量名拼写错误 "requset"
const requset = axios.create({
  baseURL: NEXT_PUBLIC_API_URL || NEXT_PUBLIC_SITE_URL,
});

// 请求拦截器：JWT 注入
requset.interceptors.request.use(async (config) => {
  const Authorization = getAuthorization(config.Authorization);
  if (Authorization) config.headers.Authorization = Authorization;
  return config;
});

// 响应拦截器：业务码检查 + 错误处理
requset.interceptors.response.use(
  async (response) => {
    const { code } = response.data;
    if (code !== 200) {
      await handleError(response);  // 40002-40005 登出，其余 toast
      throw response;
    }
    return response;
  },
  async (error) => {
    await handleError(error);
    return Promise.reject(error);
  },
);
```

关键行为：
- `skipErrorHandler?: boolean` 通过 TypeScript 扩展 InternalAxiosRequestConfig
- `handleError` 内部通过 `getTranslations("common")` 做 i18n 错误提示
- 40002-40005 错误码 → 强制调用 `Logout()`
- 非浏览器环境（SSR）跳过 toast

### 1.8 现有问题汇总

| 问题 | 影响 |
|------|------|
| 依赖外部 ppanel-docs 仓库 | API 变更无法端到端自动化 |
| @umijs/openapi 活跃度下降 | 未来维护风险 |
| 手动 queryKey 字符串 | 缓存失效 bug 风险 |
| 手动 queryFn 包装 | 每个 useQuery 多 3 行样板代码 |
| data.data?.list 双层解包 | 开发体验差，容易出错 |
| declare namespace API 全局类型 | 不够精确，无法 tree-shake |
| axios 依赖 | 额外 bundle size，不走 Next.js fetch 缓存 |
| requset 拼写错误 | 代码质量问题 |

### 2.3 client 初始化 + 拦截器（重构后）

**runtimeConfigPath 机制：** hey-api 的 `@hey-api/client-fetch` 插件在 `openapi-ts.config.ts` 中接受 `runtimeConfigPath` 参数。生成的 `client.gen.ts` 会自动 import 该文件并在创建 client 实例时调用 `createClientConfig()` 合并用户配置（如 baseUrl）。这样 baseUrl 等运行时配置不需要硬编码在生成文件中。

```typescript
// apps/admin/utils/hey-api-config.ts — runtimeConfigPath 指向此文件
// 生成的 client.gen.ts 会 import { createClientConfig } from '../../utils/hey-api-config'
// 并在 createClient() 时调用此函数合并 baseUrl 等运行时配置
import type { CreateClientConfig } from '@/services/admin/client.gen';
import { NEXT_PUBLIC_API_URL, NEXT_PUBLIC_SITE_URL } from '@/config/constants';

export const createClientConfig: CreateClientConfig = (config) => ({
  ...config,
  baseUrl: NEXT_PUBLIC_API_URL || NEXT_PUBLIC_SITE_URL,
});
```

```typescript
// apps/admin/utils/setup-clients.ts — 在 app 入口导入一次
import { client as adminClient } from '@/services/admin/client.gen';
import { client as commonClient } from '@/services/common/client.gen';
import { isBrowser } from '@workspace/ui/utils';
import { toast } from 'sonner';
import { getTranslations } from '@/locales/utils';
import { getAuthorization, Logout } from './common';

function setupClient(client: typeof adminClient) {
  // 请求拦截器：JWT 注入
  // 浏览器环境从 cookie 读取，SSR 环境无 token（SSR 需用 createSSRClient）
  client.interceptors.request.use(async (request) => {
    if (!isBrowser()) return request;
    const token = getAuthorization();
    if (token) request.headers.set('Authorization', token);
    return request;
  });

  // 响应拦截器：业务码检查 + 错误处理
  // response.clone().json() 是必需的，因为 response body 只能消费一次
  // clone() 保留原始 response 供 hey-api SDK 后续解析
  client.interceptors.response.use(async (response) => {
    if (!response.ok) return response;

    const body = await response.clone().json();
    const { code, message } = body;

    if (code !== 200) {
      if ([40002, 40003, 40004, 40005].includes(code)) {
        Logout();
        return response;
      }
      if (isBrowser()) {
        const t = await getTranslations('common');
        const msg = t(`request.${code}`) !== `request.${code}`
          ? t(`request.${code}`)
          : message;
        toast.error(msg);
      }
      throw new Error(message || `API error: ${code}`);
    }
    return response;
  });
}

setupClient(adminClient);
setupClient(commonClient);
```

```typescript
// apps/admin/utils/silent-client.ts — skipErrorHandler 替代方案
import { createClient } from '@/services/admin/client.gen';
import { createClientConfig } from './hey-api-config';
import { getAuthorization } from './common';

// 无 toast 的 client，替代 skipErrorHandler
// 复用 createClientConfig 保持 baseUrl 一致
export const silentClient = createClient(createClientConfig({}));

// 仅注入 JWT，不注册错误处理拦截器
silentClient.interceptors.request.use(async (request) => {
  const token = getAuthorization();
  if (token) request.headers.set('Authorization', token);
  return request;
});
```

```typescript
// apps/user/utils/ssr-client.ts — SSR per-request client
// 必需：layout.tsx 中的 SSR 调用（queryUserInfo、getGlobalConfig）
// 需要传入 Authorization token，不能使用全局 client（浏览器 cookie 不可用）
// 
// 现有代码 getAuthorization(config.Authorization) 通过 axios config 传 token，
// fetch 拦截器没有等价机制，所以需要 per-request client
import { createClient } from '@/services/user/client.gen';
import { NEXT_PUBLIC_API_URL } from '@/config/constants';

export function createSSRClient(token?: string) {
  const client = createClient({
    baseUrl: NEXT_PUBLIC_API_URL,
  });
  if (token) {
    client.interceptors.request.use(async (request) => {
      request.headers.set('Authorization', token);
      return request;
    });
  }
  return client;
}
```

**SSR 调用迁移示例（user app layout.tsx）：**

```typescript
// BEFORE（apps/user/app/layout.tsx）
const config = await getGlobalConfig({ skipErrorHandler: true }).then((res) => res.data.data);
const user = await queryUserInfo({ skipErrorHandler: true, Authorization }).then((res) => res.data.data);

// AFTER
import { createSSRClient } from '@/utils/ssr-client';
import { getGlobalConfig } from '@/services/common/sdk.gen';
import { queryUserInfo } from '@/services/user/sdk.gen';

const ssrClient = createSSRClient();               // 无 token，无 toast
const { data: config } = await getGlobalConfig({ client: ssrClient });

const authedClient = createSSRClient(Authorization); // 带 token，无 toast
const { data: user } = await queryUserInfo({ client: authedClient });
```

**admin app 的 layout.tsx 也有同样的 SSR 调用**（`getGlobalConfig({ skipErrorHandler: true })` + `currentUser({ skipErrorHandler: true, Authorization })`），需要在 Commit #5b 中同步迁移到 `createSSRClient` 模式。

### 2.4 API.* 兼容层（重构后）

```typescript
// apps/admin/services/_compat/api-namespace.ts
// 桥接 API.* 全局命名空间到 hey-api 具名导出
// 渐进式迁移：新代码用具名 import，旧代码通过此文件过渡

import type * as AdminTypes from '../admin/types.gen';
import type * as CommonTypes from '../common/types.gen';

declare global {
  namespace API {
    // Admin types
    type Ads = AdminTypes.Ads;
    type CreateAdsRequest = AdminTypes.CreateAdsRequest;
    type UpdateAdsRequest = AdminTypes.UpdateAdsRequest;
    type GetAdsListResponse = AdminTypes.GetAdsListResponse;
    type Response = AdminTypes.Response;
    // ... 其余类型由脚本自动生成
  }
}
```

**自动生成方式：** 此文件通过一个简单的 Node 脚本从 `types.gen.ts` 的 `export type` 声明中提取类型名，生成对应的 `declare global { namespace API { type X = AdminTypes.X; } }` 映射。脚本逻辑：

```bash
# scripts/generate-api-compat.sh（示意）
# 1. 从 types.gen.ts 提取所有 export type 名称
# 2. 生成 declare global { namespace API { type X = AdminTypes.X; ... } }
# 3. 写入 _compat/api-namespace.ts
grep "^export type" apps/admin/services/admin/types.gen.ts | \
  sed 's/export type \([A-Za-z]*\).*/    type \1 = AdminTypes.\1;/' > ...
```

该脚本在 `openapi:client` 之后自动执行，确保兼容层始终与 types.gen.ts 同步。渐进迁移完成后（所有 API.* 引用改为具名 import），删除此文件和脚本。

### 2.8 scripts/generate-openapi.sh（重构后）

```bash

set -euo pipefail
cd "$(git rev-parse --show-toplevel)"

GOPURE=server/generate/gopure-$(uname -s | tr '[:upper:]' '[:lower:]')-$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/')
OPENAPI_PLUGIN="$HOME/go/bin/goctl-openapi"
OUTDIR=docs/openapi

mkdir -p "$OUTDIR"

for spec in admin common user; do
  cd server
  # 注意：gopure 使用单横线 -plugin/-api/-dir（goctl 兼容格式）
  # spike 时需验证 gopure 版本是否支持此格式
  "../$GOPURE" api plugin \
    -plugin "$OPENAPI_PLUGIN -format json -filename ${spec}.json" \
    -api "apis/swagger_${spec}.api" \
    -dir "../$OUTDIR/"
  cd ..
done

echo "Generated: $OUTDIR/{admin,common,user}.json"
```

## Resolved Questions

1. **client-fetch vs client-axios** — 选 client-fetch。理由：零依赖、Next.js 缓存兼容、OpenCode 验证过。
2. **skipErrorHandler** — 用 createClient() 创建无 toast 的 silentClient，比 module augmentation 更干净。
3. **SSR client 单例隔离** — 用 createClient() 创建 per-request client，不共享全局单例。
4. **swagger_node.api** — 不纳入（节点通信接口，前端不调用）。
